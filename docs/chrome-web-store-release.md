# Chrome ウェブストア公開手順

このドキュメントは、`X Popular Link Filter` を Chrome ウェブストアに公開するための手順をまとめたものです。

最終確認日: `2026-03-16`

## 1. 公開前の準備

まず、審査に出せる状態まで拡張機能を整えます。

### プロジェクト確認項目

- [ ] `manifest.json` の `version` を更新する
- [ ] `manifest.json` に拡張機能アイコンを追加する
- [ ] `https://x.com/home` で正常動作することを確認する
- [ ] 対象外ページでポップアップに適切なエラーが表示されることを確認する
- [ ] 権限が最小になっているかを見直す
- [ ] UI文言が実際の挙動と一致していることを確認する

### ストア提出用に必要な素材

- [ ] `128x128` の拡張機能アイコン
- [ ] Chrome のUI用 `16x16`, `32x32`, `48x48` アイコン
- [ ] Chrome ウェブストア用スクリーンショット
- [ ] 短い説明文
- [ ] 詳細説明文
- [ ] サポートURL
- [ ] プライバシーポリシーURL
- [ ] 審査担当者向けメモ

## 2. 拡張機能アイコンを追加する

現在の実装では、`manifest.json` に `icons` がまだ定義されていません。

推奨ファイル構成:

```text
assets/
  icon16.png
  icon32.png
  icon48.png
  icon128.png
```

推奨する manifest 設定例:

```json
{
  "icons": {
    "16": "assets/icon16.png",
    "32": "assets/icon32.png",
    "48": "assets/icon48.png",
    "128": "assets/icon128.png"
  },
  "action": {
    "default_title": "X Popular Link Filter",
    "default_popup": "popup.html",
    "default_icon": {
      "16": "assets/icon16.png",
      "32": "assets/icon32.png"
    }
  }
}
```

## 3. ストア掲載文を用意する

説明文は、実際の挙動に忠実に書くことが重要です。

### 短い説明文の例

> Xホームタイムラインから人気のある外部リンク付き投稿を抽出して、ポップアップで一覧表示します。

### 詳細説明文の例

> X Popular Link Filter は、現在開いている X のホームタイムラインを解析し、外部リンクを含む投稿を収集します。その中から、いいね・リポスト・返信数の閾値を超えた投稿だけを抽出し、拡張機能のポップアップに一覧表示します。

### プライバシー説明の例

> この拡張機能は、Xホームタイムライン上の投稿を読み取り、外部リンク付き投稿の抽出とエンゲージメント判定を行います。収集した投稿内容は外部サーバーへ送信しません。

実装を変更した場合は、掲載文も合わせて修正してください。

このリポジトリには、そのまま転用できる草案として以下も追加しています。

- `docs/store-listing.md`
- `docs/privacy-policy.md`

## 4. プライバシーポリシーを公開する

小さな拡張でも、公開前にプライバシーポリシーの公開URLを用意しておくのが安全です。

最低限含める内容:

- どのページにアクセスするか
- X上のどの情報を読むか
- ローカル保存の有無
- 外部送信の有無
- 問い合わせ先

公開先の候補:

- GitHub Pages
- 自身のWebサイト
- 外部公開可能なドキュメントページ

## 5. パッケージ化前のテスト

Chrome上で手動確認します。

1. このリポジトリを `Load unpacked` で読み込む
2. `https://x.com/home` を開く
3. 拡張機能アイコンをクリックする
4. `Collect Now` を押す
5. 以下を確認する
   - 投稿が収集される
   - 外部リンク付き投稿だけが表示される
   - いいね / リポスト / 返信数が表示される
   - 該当結果がない場合に適切なメッセージが出る
   - ホーム以外のページではエラーになる

追加で確認したい項目:

- タイムラインの読み込み量が少ないケース
- 日本語UI / 英語UI の両方
- Xの表示が遅いケース

## 6. ZIPを作成する

リポジトリのルートでZIPを作成します。

```bash
cd /path/to/x-filter
zip -r x-popular-link-filter.zip . -x "*.git*" -x "*.DS_Store"
```

今後ローカル専用ファイルを増やした場合は、不要なものをZIPに含めないようにします。

## 7. Chrome ウェブストアのデベロッパー登録

Chrome ウェブストアの Developer Dashboard でデベロッパー登録を行います。

公式上の注意点:

- 公開には初回登録料が必要

Dashboard:

- https://chrome.google.com/webstore/devconsole/

## 8. 拡張機能を提出する

登録後の流れ:

1. Developer Dashboard を開く
2. `Add new item` を選ぶ
3. `x-popular-link-filter.zip` をアップロードする
4. ストア掲載情報を入力する
5. アイコンとスクリーンショットをアップロードする
6. プライバシー項目を入力する
7. サポートURLとプライバシーポリシーURLを入力する
8. 警告やポリシー通知を確認する
9. 審査に提出する

## 9. プライバシー項目は慎重に埋める

この拡張は X のページ内容を読むため、プライバシー項目は正確である必要があります。

説明できるようにしておく内容:

- なぜ `x.com` へのアクセスが必要か
- 目視可能なタイムライン投稿を読み取っていること
- データがブラウザ外へ出るかどうか
- 広告用途や販売用途があるかどうか
- 認証やリモートコードの有無

挙動を変えた場合は、更新版の公開前にプライバシー項目も更新してください。

## 10. 審査担当者向けメモを入れる

審査担当者向けメモを入れると、レビューが通りやすくなります。

記載例:

> Xにログインした状態で https://x.com/home を開き、拡張機能アイコンをクリックして "Collect Now" を押してください。拡張機能は表示中および自動スクロールで読み込まれたタイムライン投稿を走査し、外部リンク付き投稿のうち固定閾値を超えた投稿のみをポップアップに表示します。

## 11. 公開後の確認

審査通過後:

1. すぐ公開するか、手動公開タイミングを選ぶ
2. 公開済みストアページを確認する
3. ストア版を一度インストールして、実際の動作を確認する
4. ユーザーフィードバックやポリシーメールを監視する

## 12. 更新版を出すときの手順

次回以降の更新:

1. `manifest.json` の `version` を上げる
2. ZIPを再作成する
3. 同じストアアイテムに新しいパッケージをアップロードする
4. 挙動変更があれば説明文も更新する
5. 再審査に提出する

## 13. このリポジトリで公開前にまだ必要なもの

初回提出前に、少なくとも以下が必要です。

- 拡張機能アイコン
- 公開されたプライバシーポリシーURL
- 実ブラウザでのX上の動作確認
- ストア提出用スクリーンショット
